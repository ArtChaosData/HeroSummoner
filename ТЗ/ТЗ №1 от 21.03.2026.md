# ТЗ №1 — D&D 5e Character Sheet App
**Дата:** 21.03.2026

---

## Общее

Веб-приложение (PWA, responsive). Русскоязычный UI. Только официальный контент (SRD/PHB). Хранение — IndexedDB, экспорт/импорт JSON на Google Drive / Яндекс Диск.

---

## Экран 1 — Список персонажей

### Фильтр-табы

Полоска с вкладками над сеткой карточек: **Все / Избранные ⭐ / Активные ⚔️ / Кладбище 💀 / Архив 📦**. На каждой вкладке счётчик персонажей. Архивные персонажи не отображаются во вкладке «Все» — только в «Архиве».

Порядок отображения внутри любой вкладки: избранные → активные → кладбище → архив.

### Карточка персонажа

Структура карточки (сверху вниз):

**Портрет** — занимает верхние ~115% ширины карточки. Изображение кадрируется с фокусом на верхнюю треть (лицо как портрет). При наведении — лёгкий зум. Поверх портрета два бейджа: уровень (золото, верхний левый угол) и HP (цветной, верхний правый: зелёный ≥ 75%, оранжевый 25–74%, красный < 25%). Градиентный оверлей снизу плавно переходит в фон карточки.

**Кнопка избранного (★/☆)** — поверх портрета, верхний правый угол. Появляется при наведении; если персонаж в избранном — всегда видна.

**Информационный блок** (4 строки):
1. Имя персонажа — крупно, шрифт Cinzel
2. Класс · Подкласс — средний размер
3. Раса
4. Предыстория — мелкий курсив

**Кнопки действий** (нижняя строка карточки):
- «▶ Открыть» — занимает всю оставшуюся ширину
- 📄 Экспорт PDF
- { } Экспорт JSON
- 🗑 Удалить

### Состояние «Мёртв» (Кладбище)

Портрет — чёрно-белый, затемнённый. Поверх портрета — иконка черепа 💀. Бейдж HP заменяется на «✝ Пал в бою». Цветовой акцент карточки — тёмно-бордовый.

### Прочее

Сетка — `auto-fill`, минимум ~230px на карточку. В конце сетки всегда карточка «＋ Создать персонажа» (отображается только во вкладках «Все» и «Активные»). Если вкладка пуста — тематическое empty-state сообщение с иконкой. Строка импорта JSON — над сеткой карточек. Фолбэк при отсутствии портрета — инициалы персонажа на тёмном фоне.

## Экран 2 — Создание / редактирование персонажа

**Концепция:** единый экран, выглядящий как готовый лист персонажа, но с интерактивными элементами выбора. Все изменения отражаются мгновенно — выбрал расу, тут же видишь изменения в характеристиках, навыках, умениях. Принцип «что видишь в создании — то и видишь в игре, только без кнопок выбора».

**Что на этом экране:**

Верхняя строка — имя (ввод), класс (дропдаун), подкласс (дропдаун, зависит от класса), уровень (фиксирован на 1 при создании), раса (дропдаун), подраса (если есть), мировоззрение, имя игрока.

Характеристики — шесть значений с интерфейсом point-buy (27 очков). Счётчик потраченных очков. Рядом с каждой характеристикой сразу видны: итоговое значение с расовым бонусом, модификатор, итоговый спасбросок с профишенси. Другие системы набора статов — на потом.

Навыки — список всех навыков с чекбоксами профишенси. Доступные для выбора подсвечены (зависят от класса и предыстории). Навыки от предыстории ставятся автоматически и заблокированы от снятия. Итоговый бонус считается сразу.

Владения — автоматически заполняются из класса, расы, предыстории. Языки, доспехи, оружие, инструменты.

Боевые статы — КД (считается из ловкости + тип доспеха), инициатива, скорость, бонус мастерства. Всё авто.

HP — кость хитов класса × 1 + мод телосложения на 1-м уровне.

Заклинания — раздел появляется только у магических классов. Список заклинаний с фильтрами (уровень, школа, поиск). Выбор известных/подготавливаемых согласно механике класса. У всех магических классов видно: подготовлено или нет. Лимит подготавливаемых заклинаний считается автоматически. Тултип на каждом заклинании — название, описание, компоненты, дистанция, длительность.

Ресурсы класса — автоматически добавляются при выборе класса. Все с тултипами: описание механики и когда сбрасывается.

Снаряжение — выбор стартового набора (по классу или по предыстории).

**Отдельные экраны** (не вкладки, отдельные страницы/модалки):
- Предыстория персонажа — история, облик, черта характера, идеал, привязанность, слабость
- Аватар — загрузка изображения

## Экран 3 — Лист персонажа (игровой режим)

**Навигация:** нижний таббар на мобильном, левый сайдбар на десктопе. Все блоки сворачиваются по кнопке, состояние сохраняется.

**Шапка (всегда видна):** имя, класс, уровень, раса. Кнопки: Левел-ап, PDF, меню.

**Блок: Боевые статы** — КД, Инициатива, Скорость, Бонус мастерства, Пассивная внимательность. Только чтение, обновляются при изменении снаряжения.

**Блок: Здоровье** — Текущие / Макс / Временные ХП с большими кнопками +/−. Поле ввода числа, кнопки «Получить урон» и «Лечение». Кость хитов с остатком. Спасброски от смерти.

Кнопка «Короткий отдых» — диалог с выбором кубов хитов + авторегенерация нужных ресурсов. Тултип на кнопке: перечисление что именно восстановится у данного класса (например: «Ячейки заклинаний Колдуна»).

Кнопка «Длинный отдых» — ХП до макса, полный сброс всех ресурсов.

**Блок: Характеристики и навыки** — 6 характеристик с модификаторами. Рядом с каждым спасброском — кнопка броска кубика. Под каждой характеристикой список навыков, у каждого — кнопка броска. При нажатии — попап: «Скрытность: 1к20 **(14)** + 5 = **19**».

**Блок: Атаки** — список атак. Нажатие на атаку — попап: «Бросок атаки: 1к20 **(14)** + 7 = **21** ✓ | Урон: 1к8 **(5)** + 3 = **8**». Параметры атаки редактируемы. Кнопка добавить атаку вручную.

**Блок: Ресурсы класса** — счётчики с названием, текущим / максимумом, кнопками + / −. На каждом ресурсе — тултип с описанием механики и условий сброса.

**Блок: Заклинания** — список с фильтром по уровню. Счётчики ячеек по уровням (трата при применении). У каждого заклинания — тултип с полным описанием. Если заклинание требует **спасброска от противника** — явно указано в тултипе и на карточке (иконка/пометка «Спасбросок: CON СЛ 14»). Если требует **броска атаки** — кнопка броска прямо на карточке. Активная концентрация подсвечена; при получении урона — алерт «Проверь концентрацию: СЛ X».

**Блок: Инвентарь** — список предметов, количество, вес, кнопки + / −. Итоговый вес и порог перегрузки. Деньги (ПМ/ЗМ/ЭМ/СМ/ММ) с конвертацией.

**Блок: Умения и черты** — текст. Сворачивается, только чтение.

**Блок: Заметки** — свободный текст, редактируемый.

## Левел-ап (диалог)

Шаг 1: уровень +1, выбор ХП (бросок кубика с попапом или фиксированное значение).
Шаг 2: новые умения класса (показываются автоматически).
Шаг 3: новые заклинания если положено.
Шаг 4: ASI или черта (на 4, 8, 12... уровнях).
Шаг 5: подкласс, если открывается на этом уровне.

Всё в одном модальном диалоге.

---

## PDF-экспорт

Кнопка доступна и из списка персонажей, и из игрового экрана. Генерирует стандартный двухстраничный лист персонажа D&D 5e с заполненными полями.

## Nice-to-have: Лог бросков

Панель, выезжающая по кнопке — все броски с временными метками и деталями.
