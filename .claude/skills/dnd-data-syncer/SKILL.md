# Skill: dnd-data-syncer
# Goal: Синхронизация данных с dnd.su с поддержкой мульти-редакций (5e14/5e24).

## 🛠 Алгоритм работы
1. **Extraction**: Используй `WebFetch` для получения HTML. Извлекай только основной контент (id="content").
2. **Normalization**: Преобразуй данные в JSON. 
   - **Почему**: Поля `choice_lists` и `is_optional` критичны для динамического UI.
   - **Правило htlfrwbq**: Храни разные редакции в разщных разделах дата-архитектуры
3. **Validation**: Сверь результат по чек-листу ниже.

## ✅ Чек-лист валидации
- [ ] Поля `id`, `name`, `source_id`, `editions` присутствуют.
- [ ] Опциональный контент (TCE/XGE) помечен `is_optional: true`.
- [ ] Тексты описаний очищены от HTML-тегов и мусора.
- [ ] Ссылка на источник с dnd.su добавлена в метаданные.

## ⚠️ Ограничение
**Никогда не удаляй существующие данные в JSON**, если они помечены другой редакцией. Добавляй новые объекты в массив `editions`.
