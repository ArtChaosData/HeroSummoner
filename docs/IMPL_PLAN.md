### Задача 2: `buildSpellsStep` — реализовать шаг 4.4.6 «Заклинания»
**Файл:** `js/screens/create-new.js`
**Статус:** ✅ DONE
**Что сделать** (по ТЗ 4.4.6):

**Блок ① Паспорт магии** (класс-карточка, read-only)
- Иконка + название класса
- Бейдж типа касты: `Известные` / `Подготовленные` / `Пакт` / `Книга`
- Ключевая характеристика: INT/WIS/CHA + пояснение что это даёт
- Флаворный текст из `SPELL_FLAVOR` (данные встроены в функцию)

**Блок ② Числовая шапка** (компактная строка)
- «Кантрипов: N · Известных: N · Слотов 1 ур.: N»

**Блок ③ Кантрипы** (только для классов с кантрипами)
- Карточки: название + школа + описание одной строкой
- Счётчик «X / N выбрано», лишние блокируются
- Сохранение в `st.mecSpellsCantrips: []`

**Блок ④ Заклинания 1 уровня** — режим зависит от типа касты:
- **known** (Бард, Следопыт, Чародей): выбрать ровно N
- **prepared** (Жрец, Друид, Паладин, Изобретатель): опционально; показать формулу
- **book** (Волшебник): выбрать 6 в книгу, пометить 2 подготовленными
- **pact** (Колдун): выбрать 2
- Сохранение в `st.mecSpellsLevel1: []`, `st.mecSpellsBook: []`, `st.mecSpellsPrepared: []`

**Блок ⑤ Особенности касты** (read-only карточки)
- Ритуальное колдовство (если есть)
- Заклинательная фокусировка

**Кнопка «Далее»** → `goMech('equipment')`
