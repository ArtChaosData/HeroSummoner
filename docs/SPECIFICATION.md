# HeroSummoner — Спецификация продукта (ТЗ)
**Версия:** 0.3
**Дата:** 2026-03-31
**Статус:** 🟡 В работе (поэкранное обсуждение)

---

## 1. Концепция продукта

**HeroSummoner** — веб-приложение для создания, хранения, игры и прокачки персонажей D&D 5e.

| Параметр | Значение |
|---|---|
| Платформа | Веб, приоритет ПК; вёрстка учитывает удобство игры с мобильника |
| Хранилище | ⚠️ TBD — требует отдельного обсуждения (IndexedDB отклонён) |
| Аудитория | Новички + опытные игроки |
| Редакции | D&D 5e 2014 (5e14) и D&D 5e 2024 (5e24) |
| Язык | Русский |

---

## 2. Архитектура данных

### 2.1 Модель персонажа (Character)

```json
{
  "id": "uuid",
  "edition": "5e14 | 5e24",
  "name": "Строка",
  "race": "Строка",
  "subrace": "Строка | null",
  "class": "Строка",
  "subclass": "Строка | null",
  "background": "Строка",
  "alignment": "Строка",
  "level": 1,
  "stats": { "str":10, "dex":10, "con":10, "int":10, "wis":10, "cha":10 },
  "skills": ["Строка"],
  "hp": { "current":10, "max":10 },
  "_wizardState": { ... },
  "_features": { ... },
  "_spells": { ... }
}
```

**Ключевые принципы:**
- Поле `edition` определяет набор данных для всего персонажа.
- Все данные классов/рас тегируются `source_edition: "5e14" | "5e24"`.
- Особенности с выборами имеют `has_choice: true` и `choice_options: []`.
- Магические классы имеют `spellcasting_type: "known" | "prepared" | "pact"`.

### 📋 Экран 4.1: Hub (Менеджмент персонажей)
**Маршрут:** `#/`
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

#### Назначение
Главный экран приложения. Список всех персонажей, точка входа в создание, лист и управление.

---

#### 4.1.1 Табы (фильтры)

| Таб | Иконка | Что показывает |
|---|---|---|
| Все | 📜 | Все персы, кроме Архива и Черновиков |
| Избранные | ⭐ | `favorite: true` |
| Активные | ⚔️ | `status: "active"`, не избранные |
| Кладбище | 💀 | `status: "dead"` |
| Архив | 📦 | `status: "archive"` + черновики (`status: "draft"`) |

**Фильтр редакций** (над табами или рядом): Все / 5e·2014 / 5.5·2024

---

#### 4.1.2 Сортировка внутри таба

Автоматический порядок в табе «Все»:
1. Избранные (⭐) — сверху
2. Активные (⚔️)
3. Остальные (нейтральный статус)
4. Мёртвые (💀) — снизу

Внутри каждой группы: по имени (А → Я).

⚠️ **Открытый вопрос (обсудить позже):** Блок «Недавние» — персонажи, с которыми взаимодействовали последними, выводить отдельной секцией сверху?

#### 4.1.3 Карточка персонажа

**Размер / сетка:**
- CSS `auto-fill` с минимальной шириной карточки (~250–280px)
- На широком экране: 4–5 карточек в ряду (на откуп дизайнеру)
- На мобильном: 1–2 карточки

**Содержимое карточки:**
- **Портрет / аватар** (верхняя часть, portrait-crop):
  - Загрузка файла с устройства
  - Fallback: инициалы + цвет класса
  - Мёртвый: монохромный оверлей + 💀
- **Бейджи поверх портрета:**
  - Уровень (`Ур. N`)
  - HP (`текущие / максимальные`) — цвет по состоянию: зелёный / жёлтый / красный / ✝ Пал в бою
  - Редакция (`5e` / `5.5`)
  - Черновик (если `status: "draft"`)
- **Кнопка избранного** (★/☆) — поверх портрета
- **Блок информации:**
  - Имя персонажа
  - Класс · Подкласс
  - Раса · Подраса (если есть)
  - Предыстория

**Кнопки действий (всегда видны, горизонтальная строка снизу карточки):**

| Кнопка | Иконка | Действие |
|---|---|---|
| Открыть / Редактировать | ▶ / ✎ | `#/sheet/:id` или `#/edit/:id` (черновик) |
| PDF | 📄 | Скачать PDF-лист (v1: заглушка) |
| JSON | `{ }` | Скачать JSON-файл персонажа |
| Удалить | 🗑 | Confirm → удаление из БД |

> Кнопка «Прокачать» на карточке **отсутствует** — прокачка доступна только из листа персонажа.

### ✨ Экран 4.2: Создание — Выбор редакции
**Маршрут:** `#/create/new` (шаг 0)

#### Назначение
Первый шаг виззарда. Выбор редакции для нового персонажа.

#### Механики
- Две карточки: «D&D 5e (2014)» и «D&D 5e (2024)»
- Краткое описание ключевых отличий
- Выбор сохраняется в `character.edition`
- После выбора → шаг Концепт

#### Разница 5e14 / 5e24
Этот экран и есть точка разветвления.

---

### 🧩 Экран 4.3: Создание — Концепт
**Маршрут:** `#/create/concept`

#### Назначение
Флаворная часть персонажа: кто он, а не как он играет.

#### Механики

**Поля:**
- **Имя** (текстовое поле, обязательное)
- **Раса** (выпадающий список, зависит от редакции)
  - После выбора расы → подраса (если применимо)
  - Краткое описание расы / подрасы под выбором
- **Мировоззрение** (выпадающий список, 9 вариантов)
- **Предыстория** (выпадающий список)
  - Краткое описание выбранной предыстории

**Кнопка "Далее" → Класс**

#### Разница 5e14 / 5e24
- 5e24: Расы не дают бонусов к характеристикам (вместо этого — свободный бонус +2/+1 на шаге Статы)
- 5e24: Список рас отличается (добавлены новые, изменены некоторые особенности)

---

### ⚔️ Экран 4.4: Создание — Класс
**Маршрут:** `#/create/mechanics` (вкладка Класс)

#### Назначение
Выбор класса и подкласса. Отображение ключевых особенностей класса.

#### Механики

**Выбор класса:**
- Список всех классов с иконкой, кубиком хитов, кратким описанием
- Тап → раскрытие карточки с деталями (навыки, спасброски, хитовый кубик)

**Подкласс:**
- Показывается если класс выбран (для 5e14 — обычно на 3 уровне, но в виззарде предлагаем сразу как опциональный выбор)
- Список подклассов с описанием

**Счётчик навыков:**
- Сколько навыков из списка класса нужно выбрать
- Чекбоксы с названиями навыков

#### Разница 5e14 / 5e24
- 5e24: Изменён Следопыт, переработаны некоторые особенности классов на 1 уровне
- Данные загружаются из `CLASS_DATA[edition][className]`

### 🌟 Экран 4.5: Создание — Особенности
**Маршрут:** `#/create/features`

#### Назначение
Интерактивный выбор классовых и расовых особенностей 1-го уровня, где требуется решение игрока.

#### Механики

**Логика показа:**
- Показывается только если у выбранного класса/расы есть особенности с `has_choice: true` на 1 уровне
- Каждая особенность отображается как карточка:
  - Название + краткое описание
  - Если `has_choice: true` → показываем радио-кнопки / чекбоксы с вариантами
  - Если `has_choice: false` → показываем read-only (просто информация)

**Примеры особенностей с выбором (5e14):**
- Следопыт: Любимый враг (тип существ)
- Следопыт: Природный исследователь (тип местности)
- Бард: Мастерство (два любых навыка)
- Воин: Боевой стиль (список вариантов)
- Паладин: Боевой стиль

**Структура данных особенности:**
```json
{
  "name": "Любимый враг",
  "description": "...",
  "level": 1,
  "source": "Следопыт",
  "has_choice": true,
  "choice_count": 2,
  "choice_options": ["Аберрации", "Великаны", "Дракониды", "..."],
  "is_optional": false,
  "source_edition": "5e14"
}
```

**Кнопка "Далее"** → активна только после заполнения всех обязательных выборов.

#### Разница 5e14 / 5e24
- 5e24: Некоторые особенности убраны или переработаны (например, Следопыт 5e24 не имеет Любимого врага в прежнем виде)
- Tasha's-варианты (опциональные): `is_optional: true` — показывать с переключателем "Варианты Таши"

### 🎲 Экран 4.6: Создание — Статы
**Маршрут:** `#/create/stats`

#### Назначение
Определение значений характеристик одним из методов.

#### Механики

**Методы (выбор в начале экрана):**
1. **Standard Array** (15, 14, 13, 12, 10, 8) — перетаскивание по слотам
2. **Point Buy** — пул 27 очков, диапазон 8-15
3. **Случайный** (4d6 drop lowest) — кнопка броска

**Расовые бонусы:**
- 5e14: Прибавляются автоматически по расе
- 5e24: Игрок распределяет бонус +2 и +1 вручную (или +1/+1/+1 если хочет)

**Итог:** Итоговые значения + модификаторы отображаются в реальном времени.

#### Разница 5e14 / 5e24
Логика расовых бонусов (см. выше).

---

### 🎒 Экран 4.7: Создание — Снаряжение
**Маршрут:** `#/create/equipment`

#### Назначение
Выбор начального снаряжения.

#### Механики

**Режим Standard (5e14/5e24):**
- Стандартное снаряжение по классу и предыстории
- Список с чекбоксами (если класс предлагает варианты "или/или")

**Режим Buy (опциональный):**
- Стартовое золото (формула или фиксированное значение по классу)
- Магазин с фильтрами по категории (Оружие, Доспехи, Снаряжение)
- Корзина с итогом и остатком золота

**Блок оружия:**
- Показывается только после выбора класса
- Иконки оружия (золотые SVG)
- Клик по строке = выбор (не только по иконке)

#### Разница 5e14 / 5e24
- Список доступного снаряжения может отличаться
- 5e24: Незначительные изменения в стартовом снаряжении некоторых классов

### 🔮 Экран 4.8: Создание — Магия
**Маршрут:** `#/create/magic`
**Показывается:** Только для магических классов (Бард, Волшебник, Друид, Жрец, Колдун, Паладин, Следопыт, Чародей, Изобретатель)

#### Назначение
Выбор кантрипов, известных/подготовленных заклинаний 1-го уровня.

#### Механики

**Шапка экрана (информационный блок):**
- Тип касты: Известные / Подготовленные / Пакт
- Слоты заклинаний на 1-м уровне (таблица или chips)
- Ключевая характеристика заклинаний (INT/WIS/CHA)

**Раздел Кантрипы:**
- Список кантрипов класса
- Игрок выбирает N кантрипов (N зависит от класса)
- Каждый кантрип — карточка с названием и кратким описанием

**Раздел Заклинания 1-го уровня:**
- Список заклинаний 1-го уровня класса (с фильтром по школе)
- Игрок выбирает N заклинаний (или все для "prepared"-кастеров)
- Для Druид/Жрец/Паладин: показываем формулу подготовленных = модификатор + уровень

**Раздел Особенности касты:**
- Read-only: Ритуальное колдовство, Заклинательная фокусировка и т.д.

**Структура `spellcasting_type`:**

| Класс | Тип | Кантрипы 1 ур. | Заклинания 1 ур. |
|---|---|---|---|
| Бард | known | 2 | 4 |
| Волшебник | prepared (книга) | 3 | 6 в книге, 2 подготовлено |
| Друид | prepared | 2 | WIS мод + ур. |
| Жрец | prepared | 3 | WIS мод + ур. |
| Колдун | pact | 2 | 2 |
| Паладин | prepared | — | CHA мод + пол. ур. |
| Следопыт | known | — | 2 |
| Чародей | known | 4 | 2 |
| Изобретатель | prepared | 2 | INT мод + пол. ур. |

#### Разница 5e14 / 5e24
- 5e24: Следопыт получает заклинания с 1 уровня (в 5e14 — со 2-го)
- 5e24: Паладин получает кантрипы (в 5e14 — нет)
- 5e24: Некоторые заклинания переработаны

### 📄 Экран 4.9: Финальный экран создания
**Маршрут:** `#/create/final`

#### Назначение
Сводка по персонажу перед сохранением.

#### Механики

- Имя, раса, класс, предыстория, мировоззрение, уровень
- Статы и модификаторы
- HP максимальные (по классу + мод тела)
- AC (по снаряжению)
- Список навыков
- Выбранное оружие и снаряжение (блок показывается только после выбора класса)
- Список кантрипов и заклинаний (если маг)
- Список выбранных особенностей

**Кнопка "Создать персонажа"** → сохраняет в IndexedDB → редирект на `#/sheet/:id`

### 🛡️ Экран 4.10: Лист персонажа (Игровой режим)
**Маршрут:** `#/sheet/:id`

#### Назначение
Основной игровой экран во время сессии.

#### Механики

**Шапка:**
- Имя, класс, уровень, раса, бейдж редакции

**Блок HP:**
- Текущие HP / Максимальные HP
- Кнопки +/- (быстрое изменение)
- Поле ввода прямого значения
- Визуальный индикатор состояния (зелёный / жёлтый / красный / мёртв)

**Блок Защита:**
- КД (Armour Class) — рассчитывается автоматически по снаряжению
- Инициатива (+DEX мод)
- Скорость (по расе)

**Блок Характеристики:**
- 6 атрибутов с модификаторами
- Бонус мастерства по уровню

**Блок Спасброски:**
- 6 спасбросков, профессиональные отмечены
- Нажатие на спасбросок → показ значения (расчёт)

**Блок Навыки:**
- Список всех навыков с модификаторами
- Иконка профессии / экспертизы

**Блок Атаки:**
- Оружие из снаряжения
- Название, бонус атаки, урон, тип урона

**Блок Заклинания (если маг):**
- Слоты по уровням (трекер использованных)
- Список подготовленных / известных заклинаний

**Кнопка "Прокачать" (если есть нераспределённый опыт / нажата вручную)**

#### Разница 5e14 / 5e24
- Некоторые расчёты (AC, скорость, особенности) зависят от редакции персонажа

### ⬆️ Экран 4.11: Прокачка (Levelup)
**Маршрут:** `#/levelup/:id`

#### Назначение
Мини-виззард: показывает только то, что меняется при переходе с уровня N на N+1.

#### Механики

**Шаг 1: HP**
- Выбор метода: Фиксированное значение (половина кубика +1) или Бросок (анимация)
- Добавляется мод тела

**Шаг 2: Новые особенности уровня**
- Список новых особенностей (read-only)
- Если есть выборы (`has_choice: true`) → интерактивный выбор

**Шаг 3: Повышение Характеристик (ASI) — если уровень 4, 8, 12...**
- Выбор: ASI (+2 к одной или +1+1 к двум) или Фит
- Если Фит → выбор из списка фитов

**Шаг 4: Заклинания (если маг, на уровне, когда открываются новые слоты)**
- Добавить известное/подготовленное заклинание
- Заменить заклинание (для known-кастеров)

**Финал:** Сводка изменений → Кнопка "Применить"

#### Разница 5e14 / 5e24
- Расписание ASI отличается (5e24 даёт ASI на 4, 8, 12, 16, 19; 5e14 — на 4, 8, 12, 16, 19 — идентично для большинства, но Воин и Плут имеют дополнительные)
- Список фитов различается
