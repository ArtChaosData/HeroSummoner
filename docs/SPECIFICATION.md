## 1. Концепция продукта

**HeroSummoner** — веб-приложение для создания, хранения, игры и прокачки персонажей D&D 5e.

| Параметр | Значение |
|---|---|
| Платформа | Веб, приоритет ПК; вёрстка учитывает удобство игры с мобильника |
| Хранилище | Локально в браузере + синхронизация с Google Drive (⚠️ локальный движок TBD — обсудить отдельно) |
| Авторизация | Google OAuth 2.0 (опционально — для синхронизации) |
| Аудитория | Новички + опытные игроки |
| Редакции | D&D 5e 2014 (5e14) и D&D 5e 2024 (5e24) |
| Язык | Русский |

## 2. Архитектура данных

### 2.1 Модель персонажа (Character)

```json
{
  "id": "uuid",
  "edition": "5e14 | 5e24",
  "name": "Строка",
  "race": "Строка",
  "subrace": "Строка | null",
  "class": "Строка",
  "subclass": "Строка | null",
  "background": "Строка",
  "alignment": "Строка",
  "level": 1,
  "stats": { "str":10, "dex":10, "con":10, "int":10, "wis":10, "cha":10 },
  "skills": ["Строка"],
  "hp": { "current":10, "max":10 },
  "_wizardState": { ... },
  "_features": { ... },
  "_spells": { ... }
}
```

**Ключевые принципы:**
- Поле `edition` определяет набор данных для всего персонажа.
- Все данные классов/рас тегируются `source_edition: "5e14" | "5e24"`.
- Особенности с выборами имеют `has_choice: true` и `choice_options: []`.
- Магические классы имеют `spellcasting_type: "known" | "prepared" | "pact"`.

### 📋 Экран 4.1: Hub (Менеджмент персонажей)
**Маршрут:** `#/`
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

#### Назначение
Главный экран приложения. Список всех персонажей, точка входа в создание, лист и управление.

---

#### 4.1.1 Табы (фильтры)

| Таб | Иконка | Что показывает |
|---|---|---|
| Все | 📜 | Все персы, кроме Архива и Черновиков |
| Избранные | ⭐ | `favorite: true` |
| Активные | ⚔️ | `status: "active"`, не избранные |
| Кладбище | 💀 | `status: "dead"` |
| Архив | 📦 | `status: "archive"` + черновики (`status: "draft"`) |

**Фильтр редакций** (над табами или рядом): Все / 5e·2014 / 5.5·2024

---

#### 4.1.2 Сортировка внутри таба

Автоматический порядок в табе «Все»:
1. Избранные (⭐) — сверху
2. Активные (⚔️)
3. Остальные (нейтральный статус)
4. Мёртвые (💀) — снизу

Внутри каждой группы: по имени (А → Я).

⚠️ **Открытый вопрос (обсудить позже):** Блок «Недавние» — персонажи, с которыми взаимодействовали последними, выводить отдельной секцией сверху?

#### 4.1.3 Карточка персонажа

**Размер / сетка:**
- CSS `auto-fill` с минимальной шириной карточки (~250–280px)
- На широком экране: 4–5 карточек в ряду (на откуп дизайнеру)
- На мобильном: 1–2 карточки

**Содержимое карточки:**
- **Портрет / аватар** (верхняя часть, portrait-crop):
  - Загрузка файла с устройства
  - Fallback: инициалы + цвет класса
  - Мёртвый: монохромный оверлей + 💀
- **Бейджи поверх портрета:**
  - Уровень (`Ур. N`)
  - HP (`текущие / максимальные`) — цвет по состоянию: зелёный / жёлтый / красный / ✝ Пал в бою
  - Редакция (`5e` / `5.5`)
  - Черновик (если `status: "draft"`)
- **Кнопка избранного** (★/☆) — поверх портрета
- **Блок информации:**
  - Имя персонажа
  - Класс · Подкласс
  - Раса · Подраса (если есть)
  - Предыстория

**Кнопки действий (всегда видны, горизонтальная строка снизу карточки):**

| Кнопка | Иконка | Действие |
|---|---|---|
| Открыть / Редактировать | ▶ / ✎ | `#/sheet/:id` или `#/edit/:id` (черновик) |
| PDF | 📄 | Скачать PDF-лист (v1: заглушка) |
| JSON | `{ }` | Скачать JSON-файл персонажа |
| Удалить | 🗑 | Confirm → удаление из БД |

> Кнопка «Прокачать» на карточке **отсутствует** — прокачка доступна только из листа персонажа.

#### 4.1.4 Карточка «Создать персонажа»

- Визуально выделенная карточка в той же сетке (+ или FAB — на откуп дизайнеру)
- Клик → `#/create` — **сразу создаёт запись в БД со статусом `draft`** и переходит на лендинг виззарда
- Черновик немедленно появляется в Hub (таб «Архив»), чтобы не потерять прогресс

### Общая структура виззарда

```
[Hub] → "Создать" → [Landing /create]
                         ├── [Концепт /create/concept]   ← флавор
                         └── [Механика /create/mechanics] ← линейный суб-виззард
                                  edition → class → race → background
                                  → stats → spells* → equipment → final
                                  (* только для магических классов)
```

**Ключевые принципы:**
- Концепт и Механика — **нелинейны**: игрок сам выбирает с чего начать
- Внутри Механики — **линейный** прогресс-бар со шагами
- Персонаж создаётся в БД **сразу** со статусом `draft` при входе на лендинг
- Черновик автоматически отображается в Hub (таб «Архив») — никакого localStorage

### ✨ Экран 4.2: Создание — Лендинг (точка входа)
**Маршрут:** `#/create` (новый) / `#/edit/:id` (редактирование)
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

#### Назначение
Хаб создания. Показывает статус двух блоков и позволяет переходить в любой из них.

#### Механики

**При первом открытии (`#/create`):**
- Немедленно создаётся запись в БД: `{ id, status: "draft", name: "" }`
- Редирект на `#/create/:id`

**Содержимое экрана:**
- Большая кнопка **«Концепт»**
  - Иконка + заголовок + подзаголовок («Внешность, история, характер»)
  - Статус-бейдж: ✅ Готово / ⬜ Не заполнен
  - Клик → `#/create/:id/concept`
- Большая кнопка **«Механика»**
  - Иконка + заголовок + подзаголовок («Класс, характеристики, умения»)
  - Статус-бейдж: ✅ Готово / ⬜ Не заполнен
  - Клик → `#/create/:id/mechanics`
- **CTA-кнопка** (всегда видна внизу):
  - Оба блока готовы → **«Призвать героя»** (primary, активна)
  - Не всё готово → **«Сохранить и выйти»** (ghost) — черновик уже в БД, просто возврат в Hub

#### Разница 5e14 / 5e24
Редакция выбирается на первом шаге Механики (edition step).

### 🧩 Экран 4.3: Создание — Концепт
**Маршрут:** `#/create/:id/concept`
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

#### Назначение
Флавор персонажа: кто он, а не как он играет. Заполняется в любой момент, независимо от Механики.

#### Механики

**Секция «Идентичность»** (обязательные поля):
- **Имя персонажа** — текст, обязательное
- **Имя игрока** — текст, обязательное
- **Мировоззрение** — выпадающий список (9 вариантов PHB)

**Секция «Внешность»** (обязательная):
- Возраст, Рост, Вес — числовые поля
- Цвет глаз, Цвет волос, Цвет кожи — текстовые поля

**Секция «Личность»** (обязательная, Big 4 PHB):
- Личностные черты (textarea)
- Идеалы (textarea)
- Привязанности (textarea)
- Слабости (textarea)

**Секция «История»** (обязательная):
- Предыстория — textarea (свободный текст, НЕ выбор из списка — выбор предыстории в Механике)

**Необязательные поля** (опционально, можно скрыть):
- Союзники и организации
- Сокровища и дополнительное снаряжение

**Кнопка «Готово» → возврат на Лендинг** (статус Концепта меняется на ✅)

**Автосохранение:** при потере фокуса каждого поля → `DB.put(char)`

#### Разница 5e14 / 5e24
Нет разницы — концепт один и тот же для обеих редакций.

### ⚙️ Шаг 4.4.1: Редакция и Дополнения
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

Шаг состоит из двух секций: выбор редакции и выбор дополнений.

---

#### Секция A: Выбор редакции

Две карточки-переключателя:

| Карточка | Подзаголовок | Описание |
|---|---|---|
| **5e · 2014** | Классическая редакция | Проверенная временем система. Большинство обучающих материалов, русскоязычных переводов и сообщество заточены под неё. |
| **5.5 · 2024** | Обновлённая редакция | Переработанные классы, новый подход к расам (Species) и предысториям. Совместима с материалами 2014 с оговорками. |

- Выбор сохраняется в `character.edition: "5e14" | "5e24"`
- Одна карточка всегда выбрана (default: 5e14)
- Смена редакции на этом шаге сбрасывает все последующие шаги (класс, раса и т.д.) — показывается предупреждение если шаги уже заполнены

---

#### Секция B: Дополнения и справочники

**Назначение:** расширить пул доступных рас, классов, заклинаний, снаряжения и механик из официальных книг сверх базового PHB.

**UX-паттерн: Категории + строки + модалка подробностей**

Структура отображения:
```
[Категория: Классические расширения]
  ☐  Xanathar's Guide to Everything   +заклинания, подклассы, расы   [подробнее]
  ☐  Tasha's Cauldron of Everything   +подклассы, опц. правила        [подробнее ↗]

[Категория: Новые расы и виды]
  ☐  Volo's Guide to Monsters         +расы                           [подробнее]
  ☐  Mordenkainen's Tome of Foes      +расы, лор                      [подробнее]

[Категория: Сеттинги]
  ☐  Spelljammer: Adventures in Space  +расы, механики, сеттинг       [подробнее]
  ☐  Eberron: Rising from the Last War +расы, подклассы, сеттинг      [подробнее]
  ...
```

**Правила отображения:**
- Категории — всегда видны (не скрываются accordion-ом)
- Каждый доп — строка: `[чекбокс]  [Название]  [теги что добавляет]  [кнопка «подробнее»]`
- Теги добавляемого контента: `+расы`, `+заклинания`, `+подклассы`, `+механики`, `+снаряжение`, `+сеттинг`
- Кнопка «подробнее» открывает **модалку** (универсально для ПК и мобиле)

**Модалка дополнения** содержит:
- Название и обложка (если есть CC0-совместимый арт)
- Описание в 3–5 предложениях
- Список что конкретно добавляется (по типам контента)
- Предупреждения/оговорки (см. ниже)
- Кнопка «Добавить» / «Убрать» прямо в модалке

#### Категории дополнений и копирайтинг

| Категория | Что включает | Тональность описания |
|---|---|---|
| **Классические расширения** | Xanathar's, Tasha's, Mordenkainen's Monsters | Нейтральная / позитивная |

- **Любой сеттинговый доп** — пометка `🗺️ Сеттинг-специфично` в строке тегов

---

#### Данные в JSON

Каждый доп описывается в `SOURCES_DATA`:

```json
{
  "id": "xge",
  "name": "Xanathar's Guide to Everything",
  "short": "XGE",
  "category": "classic_expansion",
  "compatible_editions": ["5e14", "5e24"],
  "tags": ["spells", "subclasses", "races"],
  "is_optional": true,
  "source_id": "xge",
  "warning": null
}
```

```json
{
  "id": "tcoe",
  "name": "Tasha's Cauldron of Everything",
  "short": "TCE",
  "category": "classic_expansion",
  "compatible_editions": ["5e14", "5e24"],
  "tags": ["subclasses", "optional_rules", "spells"],
  "is_optional": true,
  "source_id": "tcoe",
  "warning": "optional_class_features"
}
```

Выбранные допы сохраняются в `character.sources: ["xge", "tcoe", ...]`

Весь контент из допов тегируется `source_id` — это позволяет фильтровать списки рас/классов/заклинаний на последующих шагах по активным источникам.

Секция раскрыта по умолчанию — в ней могут быть обязательные выборы.

##### ⑤ Подкласс

⚠️ **Два режима в зависимости от класса:**

**Режим A — Обязательный выбор на 1 уровне** (Жрец, Чародей, Колдун):

Секция **раскрыта и обязательна**, помечена:
> «Это ключевой выбор, который определяет вашу магию с первого уровня»

- Колдун → «Выберите Покровителя» (с кем заключён договор)

Для каждого варианта: название + **флаворное описание на 1–2 предложения без механик** + тег источника (PHB / XGtE / TCE) если активирован соответствующий доп.

Кнопка «Далее» заблокирована пока подкласс не выбран.

**Режим B — Опциональный предпросмотр** (все остальные классы, подкласс на 3 уровне):

Секция collapsed, заголовок: «Уже думаете о специализации? ▾»

При раскрытии:
> «Подкласс обычно выбирается на 3-м уровне. Здесь можно наметить направление — это ни к чему не обязывает.»

- Список подклассов с названием + флаворное описание
- Подклассы из допов: видны только если источник активен, помечены тегом `[XGtE]` / `[TCE]`
- Выбор опциональный

### 🌿 Шаг 4.4.3: Раса
**Статус ТЗ:** ✅ Утверждён (2026-04-01)

---

#### Layout

Тот же двухколоночный паттерн что и Класс: список слева, детальная панель справа.

---

#### Левая колонка: список рас

Расы **сгруппированы по источнику** (PHB первый, затем активные допы). Внутри каждой группы — алфавитный порядок.

Заголовок группы — кнопка-toggle (collapsible):
```
▾ PHB — Player's Handbook
  Дварф          +2 КОН
  Драконид       +2 СИЛ  +1 ХАР
  ...
▾ VGtM — Volo's Guide to Monsters
  Аасимар        +2 ХАР
  ...
```

- Аббревиатура книги + полное название в тултипе
- **ASI-бонусы** прямо в строке (+2 КОН, +1 МДР и т.п.) — для быстрого сканирования
- Расы, чьи ASI-бонусы совпадают с ключевыми статами выбранного класса — помечены звёздочкой `★` и поднимаются в топ внутри группы; пример: для Паладина (СИЛ + ХАР) помечаются Драконид (+2 СИЛ +1 ХАР), Полуэльф (+2 ХАР) и т.п.
- 5e24 / TCE Custom Origin: вместо фиксированных бонусов пишется `+2/+1 ваш выбор` — все расы равноценны, ★ не показываются
- Выбранная раса — highlighted; клик обновляет правую колонку

---

#### Правая колонка: детальная панель

Панель обновляется в два этапа: сначала при выборе расы, затем при выборе подрасы.

##### ① Шапка (обновляется при выборе расы)

- Название расы + badge источника (тултип с полным именем книги)
- **Одна строка описания** — самая суть без деталей: «Стойкий подземный народ-кузнецы»
- Контекстная подсказка, если ASI подходит под класс игрока: `★ Хороший выбор для вашего Паладина — бонусы к Силе и Харизме` (только 5e14; в 5e24 не показывается)

##### ② Подраса (если есть — сразу после шапки, до любых деталей)

Если у расы есть подрасы — они показываются **немедленно**, крупными кнопками-карточками:

```
┌─────────────────┐  ┌─────────────────┐
│    Горный       │  │    Холмовой     │
│   +2 КОН +2 СИЛ│  │   +2 КОН +1 МДР│
└─────────────────┘  └─────────────────┘
```

- Каждая карточка подрасы: название + её собственные ASI-бонусы
- При выборе подрасы — карточка выделяется, секции ③ и ④ ниже обновляются под подрасу
- Если подрас нет — секция пропускается, сразу ③

##### ③ Описание и бонусы (обновляется при выборе подрасы, если она есть)

Источник текста и бонусов:
- Есть выбранная подраса → описание и ASI берутся от **подрасы**
- Нет подрас → описание и ASI от расы

**Описание:** 2–3 предложения про конкретную подрасу (или расу без подрас) — plain language, без механических терминов.

**ASI-бонусы:**

_5e14 (фиксированные):_
```
Телосложение +2   Сила +2
«Твой персонаж будет крепче и сильнее среднего»
```
Название характеристики + величина бонуса + краткий plain-language комментарий.

_5e14 + TCE Custom Origin (переключатель):_
- `● Стандарт` — фиксированные бонусы (выше)
- `○ Custom Origin (TCE)` — «+2 к одной и +1 к другой характеристике на ваш выбор. Распределение — на шаге Характеристики.»

Пометка: «Опциональное правило Tasha's — уточни у Мастера.»

_5e24:_ «Бонусы к характеристикам выбираются свободно на шаге Характеристики — раса на них не влияет.»

##### ④ Расовые умения

Карточки read-only — список умений **выбранной подрасы + базовые умения расы**:

- Название умения + краткое plain-language описание (без чисел и формул там где можно)
- Примеры: «Тёмное зрение — видит в полной темноте на 18 м», «Устойчивость к яду — с трудом отравляется», «Псевдодракон — обнаруживает магию поблизости»
- Секция раскрыта по умолчанию (не collapsed) — это ключевая инфа для выбора

Данные из `RACE_TRAITS_DATA[raceName][subraceOrBase]` (новый файл данных).

#### Кнопка «Далее»

Активна когда:
1. Раса выбрана
2. Подраса выбрана (если она у расы есть)

→ Шаг Предыстория

#### Данные

- `character.race` — название расы
- `character.subrace` — подраса или `""`
- `character.race_source` — id источника (`"PHB"`, `"VGtM"` и т.д.)
- `character.race_asi_mode` — `"fixed"` (стандарт) | `"custom"` (TCE Custom Origin, только 5e14)

### 📜 Шаг 4.4.4: Предыстория

Вверху шага — однострочный онбординг для новичка:
> «Кем был ваш персонаж до приключений? Предыстория определяет навыки, стартовое снаряжение и социальные связи.»

#### Layout

Тот же двухколоночный паттерн: список слева, детальная панель справа.

---

#### Левая колонка: список предысторий

Предыстории **сгруппированы по источнику** (PHB всегда, затем активные допы). Внутри группы — алфавитный порядок. Внизу списка PHB — всегда «Собственная предыстория».

Каждая строка:

**5e14:**
```
Солдат          Атлетика · Запугивание
Мудрец          История · Магия
```

**5e24:**
```
Солдат          Атлетика · Запугивание  +2 СИЛ +1 КОН  [Черта]
Мудрец          История · Магия         +2 МДР +1 ИНТ   [Черта]
```

- Навыки прямо в строке — эксперт сканирует без клика
- В 5e24 добавляется ASI и badge черты — это теперь ключевая механика

---

#### Правая колонка: детальная панель

##### ① Шапка

- Название + badge источника
- Описание: 2–3 предложения, plain language

##### ② Что даёт (сводная таблица — всегда видна, раскрыта)

**5e14:**

| | |
|---|---|
| Навыки | Атлетика, Запугивание |
| Языки / Инструменты | *выбор* или фиксированные — см. ниже |
| Стартовое снаряжение | Краткий список предметов + стартовое золото |

**5e24 (дополнительно):**

| | |
|---|---|
| Бонус к характеристикам | +2 к [стат], +1 к [стат] |
| Черта 1-го уровня | [Название черты] — краткое описание |
| Навыки | … |
| Стартовое снаряжение | … |

Бонус к характеристикам и черта в 5e24 — выделены визуально (другой цвет блока), с пояснением для новичка:
> «В 5e24 предыстория влияет на характеристики и даёт особую черту — выбирай её осознанно.»

##### ③ Интерактивные выборы

Только те выборы, которые делаются ЗДЕСЬ (не отложены на шаг Снаряжения):

- **Язык** — dropdown: «Выберите язык» → список всех языков
  - Пояснение для новичка: «Дополнительный язык, который понимает ваш персонаж — полезен для взаимодействия с НПС»
- **Навык (только Собственная предыстория)** — выбор 2 из всех навыков

Отложенные выборы (инструменты, музыкальные инструменты, игровые наборы) — не показываются здесь, только статус-строка:
```
Ремесленный инструмент  → выбор на шаге «Снаряжение»
```

##### ④ Собственная предыстория (особый режим)

Если выбрана «Собственная предыстория» — панель раскрывается в расширенный режим:

- Выбор 2 навыков из всего списка
- Выбор 2 позиций из: языки / любые инструменты (можно комбинировать)
- Снаряжение — выбор одного готового набора из других предысторий
- Пояснение: «Если ни одна предыстория не подходит — соберите свою из доступных компонентов»

---

#### Кнопка «Далее»

Активна когда все обязательные выборы здесь сделаны (языки и т.п.).
Инструменты/инструменты — не блокируют «Далее», они отложены.

→ Шаг Характеристики

---

#### Данные

- `character.background` — название предыстории
- `character.background_source` — id источника
- `character.background_choices` — объект с выборами: языки, навыки (для custom)
- `character.skills` += навыки предыстории (объединяются с навыками класса)

#### Разница 5e14 / 5e24

- **5e14:** предыстория даёт 2 навыка + языки/инструменты + снаряжение + ролевая черта
- **5e24:** предыстория даёт 2 навыка + **фиксированные ASI (+2/+1)** + **фит 1-го уровня** + снаряжение. ASI применяются на шаге Характеристик автоматически.

### 🎲 Шаг 4.4.5: Характеристики
**Статус ТЗ:** ⬜ Ожидает обсуждения

#### Механики

**Метод (выбирается в начале шага):**
1. **Point Buy** — пул 27 очков, диапазон 8–15 (стоимость нелинейная: 14 = 7 очков, 15 = 9)
2. **Standard Array** — (15, 14, 13, 12, 10, 8) — drag-and-drop или dropdown по слотам
3. **Случайный** — 4d6 drop lowest × 6, анимация броска, затем назначение по слотам

**Расовые / Предысторные бонусы:**
- 5e14: автоматически прибавляются по расе
- 5e24: игрок вручную распределяет +2 и +1 по двум разным характеристикам (или +1/+1/+1)

**Real-time итог:** значения + модификаторы обновляются сразу

**Кнопка «Далее»** → Заклинания (если маг) или Снаряжение

### 🔮 Шаг 4.4.6: Заклинания *(только магические классы)*
**Статус ТЗ:** ✅ Утверждён (2026-03-31)

#### Механики

---

**① Класс-карточка («Паспорт магии»)** — *всегда раскрыта, read-only*

Показывается в верхней части экрана до выбора заклинаний. Объясняет уникальную механику касты выбранного класса. Цель — понятный онбординг для новичка, быстрое напоминание для опытного.

Карточка содержит:
- **Иконка + название** класса
- **Тип касты** (badge): `Известные` / `Подготовленные` / `Пакт`
- **Ключевая характеристика**: INT / WIS / CHA — с пояснением что это даёт
- **Флаворный текст** — 2–3 предложения о том, как ЭТОТ класс относится к магии

Примеры флавора (`spellcasting_flavor` в JSON):

| Класс | Флаворный текст |
|---|---|
| **Жрец** | Твоя магия — дар бога. Всё заклинание-пул доступен тебе всегда. Каждое утро ты **заново выбираешь**, какие молитвы подготовить — можно менять хоть каждый день. |
| **Волшебник** | Ты учёный магии. Заклинания живут в **Книге заклинаний** — записанные формулы. Каждое утро ты изучаешь часть из них. Найденные свитки можно копировать в книгу. |
| **Бард** | Магия — твоё искусство. Ты знаешь небольшой набор заклинаний **наизусть** и не можешь менять их каждый день. Но на каждом новом уровне можешь **заменить** одно известное. |
| **Колдун** | Твоя сила — договор с Покровителем. Ты знаешь мало заклинаний, но твои слоты **восполняются на коротком отдыхе**. Покровитель не терпит, когда его силу используют зря. |
| **Друид** | Природа говорит с тобой. Весь пул заклинаний открыт, каждый день ты **подготавливаешь** нужные по ситуации. |
| **Паладин** | Твои заклинания — клятва, воплощённая в силе. Пул открыт, ты **подготавливаешь** заклинания каждый день. Магия приходит только с 2 уровня (5e14) / с 1 уровня (5e24). |
| **Следопыт** | Магия природы поддерживает твой путь. Небольшой набор **известных** заклинаний, которые ты не меняешь каждый день. |
| **Чародей** | Магия в твоей крови — врождённая сила. Знаешь заклинания **наизусть**, не меняешь ежедневно. Особая механика: **Очки Чародейства** для создания дополнительных слотов. |
| **Изобретатель** | Магия через изобретения и формулы. **Подготавливаешь** каждый день из открытого пула. |

Данные хранятся в `CLASS_DATA[edition][className].spellcasting_flavor: { type, stat, description }`.

---

**② Числовая шапка** (компактная строка под карточкой):
- Кантрипов: **N** · Известных/Подготовленных: **N** · Слотов 1 ур.: **N**

---

**③ Кантрипы** — выбор N из списка класса
- Карточки с названием + школа + одна строка описания
- Выбранные подсвечиваются, счётчик «X / N выбрано»

---

**④ Заклинания 1 уровня** — зависит от типа касты:
- **known** (Бард, Следопыт, Чародей): выбор ровно N заклинаний из списка
- **prepared** (Жрец, Друид, Паладин, Изобретатель): выбор опциональный; показывается формула `подготовлено = мод + уровень`; список можно пролистать
- **book** (Волшебник): выбрать 6 заклинаний в книгу, из них отметить 2 подготовленных
- **pact** (Колдун): выбор 2 заклинаний, пометка что слоты = короткий отдых

---

**⑤ Особенности касты** — read-only карточки внизу:
- Ритуальное колдовство (если есть у класса)
- Заклинательная фокусировка
- Специфические особенности (Мистические воззвания Колдуна, Метамагия Чародея — *только инфо, выбор на шаге Особенностей*)

---

| Класс | Тип | Кантрипы 1 ур. | Заклинания 1 ур. |
|---|---|---|---|
| Бард | known | 2 | 4 |
| Волшебник | book | 3 | 6 в книге, 2 подготовлено |
| Друид | prepared | 2 | WIS мод + ур. |
| Жрец | prepared | 3 | WIS мод + ур. |
| Колдун | pact | 2 | 2 |
| Паладин | prepared | — (5e14) / 2 (5e24) | CHA мод + пол. ур. |
| Следопыт | known | — (5e14) / 2 (5e24) | 2 |
| Чародей | known | 4 | 2 |
| Изобретатель | prepared | 2 | INT мод + пол. ур. |

**Кнопка «Далее»** → Снаряжение
